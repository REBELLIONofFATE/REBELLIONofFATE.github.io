创建实体类

插入数据库内的数据为对应实体类的数据，如插入数据库中student表，则需要对应实体类Student，student表中数据对应为实体类的属性。

例子如下

public class Student {
    private int id;
    private String name;
    private int age;
public Student() {
    super();
}
public Student(String name, int age) {
    this.name = name;
    this.age = age;
}
public Student(int id, String name, int age) {
    super();
    this.id = id;
    this.name = name;
    this.age = age;
}
public int getId() {
    return id;
}
public void setId(int id) {
    this.id = id;
}
public String getName() {
    return name;
}
public void setName(String name) {
    this.name = name;
}
public int getAge() {
    return age;
}
public void setAge(int age) {
    this.age = age;
}
@Override
public String toString() {
    return "Student [id=" + id + ", name=" + name + ", age=" + age + "]";
}
}

创建实体类的映射接口

实体类与其映射接口一一对应，映射接口为接下来的数据操作提供抽象方法，例如

StudentMapper.java

接口不用写修饰符，接口的默认修饰符为public。接口内的方法也不用写修饰符，且没有方法体。

创建映射文件

创建的映射文件应与映射接口同名，如StudentMapper.xml，其文件后缀为.xml

其模板如下

其中mapper标签中属性namespace的值为映射接口的全限定名。

接着在mapper标签下写所需要的sql的标签，id为映射接口对应的中抽象方法的方法的名字，例如

<insert id="insertStudent" parameterType="student" useGeneratedKeys="true">
    insert into student(name,age) values(#{name},#{age})
</insert>

其中sql语句末尾不能加标点符号

parameterType设置具体实现类的参数类型可以为基本类型也可以是类，若是类则对应值为实体类的全限定名，此处不同因为在mybatis文件中取了别名。

useGeneratedKeys=”true”为让主键自增。

resultType属性可以设置返回值，如果设置返回类型为类则为类的全限定名或别名。

#{}为取值，#{}当中应写入property值，property为属性对应set与get的方法去掉set与get后的单词首字母小写单词。

利用单元测试来执行想要运行的方法，在方法上添加注解@Test

        @Test
public void insert_test(){        
        try {
            //1.读取总的配置文件 生成InputStream对象
            InputStream inputStream = 
                    Resources.getResourceAsStream("mybatis-config.xml");    

            //  sqlSessionFactoryBuilder ------->  sqlSessionFactory------>sqlSession

            //2.获取sqlSessionFactoryBuilder
            SqlSessionFactoryBuilder sessionFactoryBuilder 
                                  = new SqlSessionFactoryBuilder();
            //3.构建sqlSessionFactory工厂
            SqlSessionFactory sqlSessionFactory = 
                      sessionFactoryBuilder.build(inputStream);

            //4.获取一个sqlSession对象
            SqlSession sqlSession = sqlSessionFactory.openSession();

            //5.通过sqlSession对象获取到映射接口的实现类对象
            //Class镜像  class类
            StudentMapper mapper = sqlSession.getMapper(StudentMapper.class);
        //6.调用插入方法的执行
        mapper.insertStudent(new Student("张三", 10));

        //7.事务处理 事务的提交
        sqlSession.commit();

    } catch (IOException e) {
        e.printStackTrace();
    }

我们需要获取mybatis-config.xml内的文件内容，然后通过方法来实现对数据的具体操纵。

首先通过getResourceAsStream方法读取地址为mybatis-config.xml文件的内容，返回值为InputStream为一个流，并不能获取文件类的具体信息，想要翻译流中的内容，我们需要一个sqlSession对象，而sqlSession对象不能直接获取，所以，我们需要先用构建sessionFactoryBuilder对象，然后用获取到的流inputStream来构建sqlSessionFactory对象，最后用sqlSessionFactory来构建sqlSession对象。

mybatis中直接提供获取实现类的对象，所以不需要我们手动继承再重写。方法sqlSession.getMapper();其中参数为Class，这个不是类，是类的镜像（抽象出java类的特点）。镜像的写法为类名 .class 。此处应该写入的是需要实现的映射接口的镜像。

mapper为生成的实现类对象，然后可以直接使用对应接口的方法操纵数据

修改数据库需要提交commit()方法，而查询不需要提交，因为查询不涉及对数据库的修改。

对封装SqlSession的封装

在用单元测试时我们必须先获取SqlSession对象，而具体对数据库的操作方法很多，但是SqlSession对象获取的方式却是一致的。所以我们可以简化这个过程，创建一个单独的函数来获取SqlSession对象，之后每次只需要调用这个方法就可以获取到我们想要的SqlSession对象。

public class MybatisSqlSessionFactoryUtil {
    //获取工厂类对象
    public static SqlSessionFactory getSqlSessionFactory() {

        SqlSessionFactory factory =null;
        try {
            InputStream inputStream = Resources.getResourceAsStream("mybatis-config.xml");
            SqlSessionFactoryBuilder builder = new SqlSessionFactoryBuilder();
             factory = builder.build(inputStream);
        } catch (Exception e) {
            e.printStackTrace();
        }
        return factory;
    }

    //调用上面静态方法获取SqlSession类对象，需要SqlSession类对象可以直接调用此方法。
    public static SqlSession getSqlSession() {

        SqlSessionFactory factory=getSqlSessionFactory();
        return factory.openSession();

    }
}

static修饰符所修饰的方法表示为静态方法，而静态方法可以不需要通过类所创建的对象来调用，可以直接使用类来调用，使用 类名.方法就可以实现对方法的调用。
